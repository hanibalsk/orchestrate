# Story 12.8: Requirements REST API

Status: ready-for-dev

## Story

As a **developer**, I want **REST endpoints for requirements management**, so that **I can integrate with other systems**.

## Acceptance Criteria

1. **AC1**: `POST /api/requirements` - Create requirement
2. **AC2**: `GET /api/requirements` - List requirements
3. **AC3**: `GET /api/requirements/:id` - Get details
4. **AC4**: `PUT /api/requirements/:id` - Update
5. **AC5**: `POST /api/requirements/:id/generate-stories`
6. **AC6**: `GET /api/requirements/:id/trace`
